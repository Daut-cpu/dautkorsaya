# dautkorsaya — Telegram video note bot

Telegram-бот на aiogram, который принимает видео и возвращает его в виде
video note (кружочка).

## Возможности

Бот показывает меню (`/start`) с двумя функциями:

* **🔵 Видео в кружок** — принимает видео (`video`) и видеофайлы, отправленные
  как документ (`document`), конвертирует через FFmpeg в квадрат 480×480 с
  обрезкой по центру и отправляет результат через `sendVideoNote`
* **⬇️ Скачать по ссылке** — скачивает видео по ссылке на Instagram или
  Facebook (через `yt-dlp`) и присылает его обратно обычным видео

Общее:

* Удаляет временные файлы после обработки
* Обрабатывает ошибки скачивания/конвертации/отправки, не роняя бота

## Требования

* Python 3.10+
* Установленный в системе `ffmpeg`

## Установка и запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# впишите в .env токен бота, полученный у @BotFather

python bot.py
```

## Запуск через Docker

FFmpeg уже встроен в образ — на хосте ничего дополнительно ставить не нужно.

```bash
cp .env.example .env
# впишите в .env токен бота, полученный у @BotFather

docker build -t video-note-bot .
docker run --rm --env-file .env video-note-bot
```

Если используешь `cookies.txt` (см. ниже), примонтируй его отдельным томом —
класть секреты внутрь образа не стоит:

```bash
docker run --rm --env-file .env -v "$(pwd)/cookies.txt:/app/cookies.txt:ro" video-note-bot
```

## Авторизация для приватного/защищённого контента (куки)

Многие посты Instagram (и часть видео Facebook) отдаются только залогиненным
пользователям — анонимный запрос `yt-dlp` в этом случае падает с ошибкой
вида `Instagram sent an empty media response`. Чтобы это обойти, можно
передать `yt-dlp` куки авторизованной сессии:

1. Установи в браузер расширение для экспорта cookies в формате Netscape,
   например «Get cookies.txt LOCALLY» (Chrome/Firefox).
2. Залогинься в Instagram/Facebook в этом браузере, открой сайт, экспортируй
   куки в файл `cookies.txt`.
3. Положи файл в корень проекта (или укажи свой путь через
   `YTDLP_COOKIES_FILE` в `.env`). Бот подхватит его автоматически, если файл
   существует — ничего больше настраивать не нужно.

Проверить файл перед использованием (формат, наличие ключевых кук
авторизации, не просрочены ли):

```bash
python3 check_cookies.py cookies.txt
```

**Важно про безопасность:**

* Файл `cookies.txt` — это фактически ключ от аккаунта на время жизни сессии.
  Он уже в `.gitignore`/`.dockerignore` — не коммить его и никому не пересылай.
* Используй отдельный (не основной) аккаунт Instagram/Facebook для этого —
  автоматизированные запросы могут привести к ограничениям или блокировке
  аккаунта со стороны площадки.
* Сессия в куках рано или поздно протухает (разлогин, смена пароля, защитные
  меры площадки) — тогда нужно будет экспортировать файл заново.

## Структура

* `bot.py` — точка входа, запуск polling
* `config.py` — конфигурация (токен, лимиты)
* `handlers.py` — обработчики сообщений и меню (FSM-переходы, скачивание, конвертация, отправка)
* `converter.py` — обёртка над FFmpeg (crop + scale в квадрат)
* `downloader.py` — обёртка над `yt-dlp` для скачивания видео по ссылке (Instagram/Facebook)
* `keyboards.py` — reply-клавиатуры (главное меню, отмена)
* `states.py` — состояния FSM для сценария «скачать по ссылке»
* `Dockerfile` — образ с Python и FFmpeg для деплоя без ручной установки зависимостей

## Примечание

Скачивание видео по ссылке предполагается для личного использования
(например, сохранить своё же видео). Соблюдай авторские права и условия
использования Instagram/Facebook при работе с чужим контентом.
